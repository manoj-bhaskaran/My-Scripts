# Module deployment configuration
# One directive per non-empty, non-comment line.
#
# Format (pipe-separated):
#   ModuleName | RelativePathFromRepoRoot | Targets | [Author] | [Description]
#
# Notes:
# - Fields 4 and 5 (Author, Description) are OPTIONAL.
# - If omitted, the deployer uses:
#       Author      -> $env:USERNAME
#       Description -> "PowerShell module"
# - Targets: comma-separated list of any of the following:
#     System         -> Deploy to C:\Program Files\WindowsPowerShell\Modules\User\<ModuleName>\<version>
#     User           -> Deploy to %USERPROFILE%\Documents\WindowsPowerShell\Modules\<ModuleName>\<version>
#     Alt:<ABS_PATH> -> Deploy to <ABS_PATH>\<ModuleName>\<version>
# - Whitespace around fields is trimmed.
# - Avoid using the '|' character inside Author/Description (no quoting/escaping supported).
#
# -------------------------
# EXAMPLES
# -------------------------
# (A) Backward-compatible: 3 fields only
#     -> Uses default Author and Description
# PostgresBackup|src\common\PostgresBackup.psm1|System
#
# (B) Provide Author only (4 fields)
# PostgresBackup|src\common\PostgresBackup.psm1|System|DBOps Team
#
# (C) Provide Author and Description (5 fields)
# PostgresBackup|src\common\PostgresBackup.psm1|System|DBOps Team|PostgreSQL backup utilities
#
# (D) Multiple targets (System + User)
# PostgresBackup|src\common\PostgresBackup.psm1|System,User|DBOps Team|Shared module for DB backups
#
# (E) Alternate base path
# AnotherModule|Modules\Another\Another.psm1|Alt:D:\Tools\PSModules|Platform Team|Internal admin tooling
#
# (F) Mixed targets with Alt
# AnotherModule|Modules\Another\Another.psm1|System,Alt:C:\Program Files\PS-SharedModules|Platform Team|Admin tooling
#
# -------------------------
